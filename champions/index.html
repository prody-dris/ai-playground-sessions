Why we start rationed — not maxed out

Starting everyone at full, unrestricted access gives you no visibility into what's actually happening. You learn nothing. You can't tell which team is using it more, which use cases are generating real value, or which users are token-efficient versus wasteful.

Starting rationed flips that. Two users doing the exact same task — say, sprint planning — but one is consuming 3× the tokens for the same output quality. That's a discovery you can only make when you're watching usage closely. That data is what shapes decisions on whether to expand seats, how to train the team, and where AI is actually working versus where it's just being used.

Rationing is not restriction — it's signal collection

This phase is about understanding, not limiting. As usage patterns stabilise and we know what's worth the investment, access expands. The champions who manage it well now will have the data to make that case.

The Daily Standup Structure

Champions decide at standup who uses Claude that day, for what, and in which time blocks.

Standup
Champion assigns the day's two active users and their work streams. Example: "Aastha on sprint notes this morning, Rahul on POC draft after lunch."
Block 1
First user works their assigned task. Saves output to shared location. Logs what they used it for (one line is enough).
Handoff
First user notifies champion: done. Champion checks output if it's going anywhere important. Clears the slot.
Block 2
Second user picks up. Same cycle — task, output, log, notify.
EOD
Champion notes the day's usage: who, what, outcome, rough token feel. This is the discovery data that matters over weeks.
03 · Usage Patterns

How to Actually Use It

AI is a thinking partner, not a search engine. These are the four patterns that give you real leverage in daily delivery and product work.

AI as Sprint Planner

Turn raw backlog items into a structured sprint in under 20 minutes.

claude.ai · Sprint Planning
You
Here are 12 items from our backlog [paste]. We have 2 weeks, 3 devs at 80% capacity. Prioritise the top 6, flag dependencies, and note anything that shouldn't be split across sprints.
Claude
Recommended sprint (6 items):
1. [Item A] — unblocked, ~3 days
2. [Item B] — depends on A completing first
3. [Item C] — independent, ~2 days
...

Flagged: Items 7 and 9 share a dependency on the auth module — don't split them.
Give Claude the capacity constraint upfront — it changes the output significantly.
Ask it to flag blockers separately. That's usually the most valuable part.
Champion reviews the sprint plan before it goes to the team.

AI as POC Builder

Validate an idea before committing sprint capacity — fast concept drafts.

claude.ai · POC Draft
You
I want to validate an idea before speccing it: automated grant eligibility check in mGrant. Help me draft a rough POC — inputs needed, core logic, and user-facing output. Keep it concept level, not implementation.
Claude
POC — Grant Eligibility Check

Inputs: Org type, FCRA status, CSR budget year, application category
Logic: Rule-based first pass → flag edge cases for manual review
Output: Eligible / Not Eligible / Needs Review + reason code

Buildable as a Frappe validation hook. Estimated: 3–5 dev days.
Frame it as "help me think through" not "build me" — better structured output.
Always include the constraint: stack, team size, timeline.
POC output goes to champion first — they decide if it warrants a formal spec.

AI as Daily Thinking Partner

Unblock yourself on decisions, articulate problems clearly, prep for calls — fast.

claude.ai · Daily Use
You
I'm about to have a call with a client frustrated about reporting delays. Help me structure my talking points: what to acknowledge, what to commit to, and what to push back on.
Claude
Acknowledge: The delay is real. Own it early.

Commit to: A specific date — not a range. One concrete action by EOW.

Push back on: Changing report format mid-cycle — that adds delay. Offer a format review after this cycle closes instead.
Use it for standup prep — paste your tasks, ask "what's blocking what?" for a 2-min clarity check.
Great for drafting difficult messages before you send them.
Start with your own view, then test it. Don't outsource thinking — accelerate it.

AI as Prompt Reviewer

Champion reviews teammate outputs like a PO reviews a ticket — before it ships.

claude.ai · Review Mode
Champion
My teammate used Claude to draft a client status update. Here it is: [paste]. Flag anything that overpromises, reveals internal process, or is factually vague.
Claude
Flagged (2 items):
1. "By next week" — too vague. Specify a date or remove.
2. "We're still finalising internally" — remove. Client doesn't need to know this.

Paragraph 3 is accurate and clear — no changes needed.
Review only applies to outputs going into decisions, client docs, or sprint plans.
If it's clean — say so and move on. The gate should take 2 minutes, not 20.
Build a "flagged patterns" list over time — what Claude consistently gets wrong in your context.

05 · Enterprise Features

What Your Account Actually Unlocks

Claude Enterprise is not just a chat upgrade. These are the features champions and their teams can actively use — most teams don't know they exist.

🏢

Organisation Skills

Provisioned by admin, available to everyone

Admin (Ravi) provisions skills org-wide — they automatically appear for all users without any setup.
Skills shape how Claude behaves for your specific context — Dhwani processes, mGrant terminology, delivery standards.
Champions can request org skills to be created for recurring use cases: grant reporting, sprint planning templates, client update formats.
Org skills persist across sessions — no re-explaining context every time.
🧑‍💻

Personal Skills & Workspace

Each user builds their own context layer

Each user can create personal skills — custom instructions, prompt templates, and role-specific context that loads automatically.
Skills can be shared with specific teammates or the whole org — champions share their best prompts as skills, not just text in a doc.
Skills are organised in three tiers: Personal (yours), Shared (from a colleague), Organisation (from admin). All visible in one place.
Build a personal skill for your most repeated task — sprint planning, status updates — and stop rewriting the same context every session.
📁

Projects

Persistent workspaces with shared context

Projects are dedicated workspaces with their own context files, working documents, and conversation history — scoped per project, not per chat.
Upload reference documents into a project — Claude reads them as background context across every conversation in that project.
Built-in RAG: when project content is large, Claude automatically expands context capacity — up to 10× normal window — to keep everything in scope.
Use a project per client engagement or per sprint — context carries forward, you don't start from scratch each time.
🔌

Integrations & Admin Controls

Governance, SSO, and external connectors

SSO and domain capture — Ravi manages access centrally, no credential sprawl.
Audit logging — admin can see what's being used, by whom, and flag misuse. This is the data layer behind the rationing model.
Connectors via MCP — Claude can pull from external systems (Google Drive, Notion, internal tools) and act within workflows. Available for future expansion.
Centralised billing with usage visibility — when champions request data on who's using what, Ravi can pull it directly from the admin dashboard.
