Session 0

Introduction & Open Build

The session that started it all. We set the format, introduced the tools, and agreed on one rule: one hour a week, we sit together and learn AI by doing AI.

What We Established

1

The format: Open dialogue, not lectures. Bring a problem, bring a question, or just work alongside the group. Informal, but consistent.

2

The tools: Claude (UI + Code), Codex, GitHub. Each has a role. We don't force one tool — we help you pick the right one.

3

MD files: Markdown is the universal format for AI. It compresses information, preserves structure, and is 10x more efficient than Word docs for AI consumption.

4

The skills concept: Domain knowledge captured in MD files that AI can reference. CSR rules, grant lifecycle, dashboard specs — your expertise, made reusable.

5

Token efficiency: Every wasted token is a thought that never happened. Write concisely, use skills to avoid repetition, always use Extended mode.

Key Decisions

AI is in trial phase. No deadlines. No mandates. We explore, we experiment, we share what works. The goal is building comfort and confidence — not shipping production code.

~

One hour per week is the only ask. Come every Thursday. If the time doesn't work, we change it together.

~

Bring a project each week. A small problem, a use case, a question. If you don't have one, Nihaan will have one ready.

~

Share your learnings. When you discover something, share it in the group. When you create a skill, share it so others benefit.

Tools Covered

Claude UIBest for thinking, drafting, brainstorming, document creation. User-friendly.
Claude CodeCLI tool. Best for multi-file code work, Frappe features, parallel tasks. Requires setup.
CodexOpenAI's coding tool. Good for Python scripts, second opinions. Free to explore.
ChatGPTStrong for complex reasoning (GPT 5.3/5.4). Good for breaking down requirements.

Model Guide

O

Opus — deep thinking, complex tasks, when you're starting from a blank slate. More expensive but more thorough.

S

Sonnet — when you know exactly what you want. Faster, cheaper, preferred by developers for coding.

H

Haiku — very short tasks, generally for developer agents. Not recommended for our use cases yet.

Always use Extended mode. It does not use extra tokens. It enables double-checking — the AI thinks twice before answering. There is no downside.

Concepts Introduced

1

Prompt guide: Assume AI is a brilliant fresher from IIT Delhi. Knows everything about tech, knows nothing about your context. Brief accordingly.

2

Skill selector: When you have many skills, a skill selector routes the AI to the right one. Like a traffic cop for your knowledge base.

3

MCPs (connectors): Connect Claude to Gmail, Slack, Calendar, GitHub, and more. Developers can also build custom connectors.

4

SDLC for PMs: Even non-developers need a process. Develop → Peer review → UAT → Production. We'll define ours together over the coming sessions.
