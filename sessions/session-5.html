Skills, Protocol & Your First Full Build.

Session 5 · Thursday, 16 April 2026 · 5:00 – 6:00 PM

Objective

Your AI gives inconsistent outputs because it doesn’t know your rules. Session 5 fixes that. We cover shared skills that make AI consistent, the Builder Protocol as your actual SDLC, and a live walkthrough of someone building a full client setup end-to-end.

What you'll walk out with

  • Understanding of what skills are and how to use them (CLI and UI paths)
  • A shared skills repository you can contribute to
  • The Builder Protocol as a build order: theme → form → dashboard
  • Real lessons from an end-to-end client setup built with AI

What We'll Cover

1

Skills Repository — Shared markdown SOPs that make AI consistent across the team. CLI path (GitHub, skills folder) and UI path (custom instructions). Why your outputs are inconsistent and how shared skills fix it.

2

Builder Protocol as SDLC — Theme → Form → Dashboard. The actual build order for delivering client work with AI. Not a framework on a page — the workflow you follow.

3

Kashish’s Journey — End-to-end client setup on Frappe, built with AI. What worked, what didn’t, what she’d do differently. Learn from her experience, not just her output.
