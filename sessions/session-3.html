Your AI Knowledge Base — Hands On.

Session 3 · Thursday, 26 March 2026 · 5:00 – 6:00 PM

Objective Today's Session

We've talked about tools, mechanics, and concepts across three sessions. Today we build. By the end of this hour, you'll have a working AI knowledge base on GitHub — one your AI can read, learn from, and use every time you start a conversation.

By 6:00 PM, you will have

  • A personal GitHub repository (your AI's home base)
  • A CLAUDE.md file (your AI's memory of you and your work)
  • Your first domain skill (CSR rules, grant lifecycle, dashboard specs — whatever you know best)
  • A working connection between GitHub and Claude UI

The Big Idea

"If you can't explain it simply, you don't understand it well enough." — Richard Feynman

Here's the problem: every time you open a new Claude conversation, it's like talking to a brilliant stranger. They know everything about technology — but nothing about you, your project, or your domain.

Today, we fix that. We give your AI a memory — stored on GitHub, accessible from anywhere, shareable with your team.

Think of it this way

When a new team member joins, you spend the first week explaining how things work. CLAUDE.md is that onboarding document — except your AI reads it in 2 seconds and never forgets it.

This Site

What you're looking at right now IS a GitHub repository. Every session page, every resource, every guide — it's all HTML and Markdown files stored on GitHub and published via GitHub Pages. No Word docs. No email attachments. One source of truth, always up to date, accessible to everyone.

This is exactly what we want each of you to build for your own AI workflows. A place where your knowledge lives, grows, and works for you — automatically.

Walkthrough

1
Set Up Your GitHub Account

Already have one? Skip to Step 2.

Need one? Go to github.com and sign up with your Dhwani email.

  • Pick a username you'll remember (your name works fine)
  • Free plan is all you need
  • Verify your email when prompted
What is GitHub?

Think of GitHub as Google Drive — but it remembers every version of every file you've ever saved. You can go back to any point in time. And multiple people can work on the same files without overwriting each other.

2
Create Your Personal Repository

A repository (repo) is just a project folder on GitHub.

  • Click the + button (top right) then New repository
  • Name it: my-ai-workspace (or anything you like)
  • Set it to Private (only you can see it)
  • Check "Add a README file"
  • Click Create repository
What just happened?

You created a folder in the cloud. It has one file (README.md). That's it. Nothing scary. This folder is now your AI's brain — and right now it's empty. Let's fill it.

3
Create Your CLAUDE.md

CLAUDE.md is the most important file in your AI workflow. It tells AI who you are, what you work on, and how you like to work.

  • In your repo, click Add file then Create new file
  • Name it: CLAUDE.md
  • Paste the template below
  • Edit it to match your role and preferences
  • Click Commit changes (green button)
# CLAUDE.md — [Your Name] ## Who I Am - Role: [Your title at Dhwani RIS] - Domain: [What you work on — grants, CSR, dashboards, MIS, etc.] - Technical level: [Be honest — "non-technical PM", "comfortable with Excel"] ## What I Work On - Products: [mGrant, Frappe LMS, dashboards, etc.] - Clients: [Who you serve, what they care about] - Current focus: [What you're working on this month] ## How I Like to Work - Be concise. Lead with the answer. - Don't ask me to write or edit code. - When in doubt, ask me — don't guess. - [Add your own preferences] ## Domain Knowledge I Have - [What do you know that the AI doesn't?] - [CSR Section 135 rules? Grant lifecycle? MIS report formats?] - [This is your superpower. Write it down.]
"The most important thing is to get started. Your first CLAUDE.md doesn't need to be perfect — it needs to exist." — Andrew Ng's principle, applied here
4
Create Your First Domain Skill

A skill is a cheat sheet you give your AI. Write it once, and AI references it forever instead of you explaining the same thing every conversation.

Why skills matter

Imagine you hire a new analyst. Day one, you explain CSR rules. Day two, you explain them again. Day three, again. That's what happens without skills — your AI starts from zero every time. A skill file is the onboarding doc that makes day one the only day you explain.

Pick your area. What do you know that AI doesn't?

A

CSR compliance — Section 135 rules, MCA reporting, budget cycles

B

Grant lifecycle — proposal to disbursement to closure

C

Dashboard specs — KPIs, data sources, filter logic

D

MIS reporting — report formats, data validation, client expectations

E

Client management — communication norms, escalation paths, SLAs

F

Something else — your expertise, your rules, your domain

Now create it:

  • In your repo, click Add file then Create new file
  • Name it: skills/[your-area].md (e.g., skills/csr-compliance.md)
  • Use the template below
  • Commit when done
# Skill: [Your Area] # Purpose: Give AI the domain knowledge it needs ## What This Covers [One paragraph: what area does this skill address?] ## Key Concepts - [Concept 1]: [Brief explanation] - [Concept 2]: [Brief explanation] - [Concept 3]: [Brief explanation] ## Rules and Constraints - [What must always be true in this domain?] - [What are the compliance requirements?] - [What mistakes do people commonly make?] ## Common Tasks - [Task 1]: [How it should be done] - [Task 2]: [How it should be done] ## Vocabulary - [Term 1] = [What it means in your context] - [Term 2] = [What it means in your context]

Start small. 10 bullet points is better than an empty file. You can always add more later. The point is to capture what's in your head so your AI can use it.

5
Connect GitHub to Claude UI

Make Claude actually read your GitHub repo. No terminal needed.

Generate a token

  • Go to github.com — click your avatar (top right) — Settings
  • Scroll down to Developer settings (bottom of left sidebar)
  • Personal access tokens — Tokens (classic)
  • Click Generate new token (classic)
  • Name: claude-access
  • Expiration: 30 days
  • Scopes: check repo (that's all you need)
  • Copy it immediately — you won't see it again

Use it in Claude

  • Open claude.ai and start a new conversation
  • Tell Claude: "I have a GitHub repo at github.com/[your-username]/my-ai-workspace. Here is my token: [paste token]. Please read my CLAUDE.md and skills folder, then tell me what you learned about me."
  • Watch it read your files and summarise your profile

What just happened? Claude read your GitHub files and now understands your role, your domain, and your preferences. Next conversation, paste the same instruction and it starts from the same baseline.

Security note: Personal access tokens are like passwords. Never share them in group chats or emails. They're for your Claude sessions only. If one leaks, revoke it from GitHub Settings immediately.

6
Build the Habit: Update After Every Session

The biggest mistake: creating CLAUDE.md once and never updating it.

After every AI session, ask Claude:

"Summarise what we did today and give me updates for my CLAUDE.md. Format it so I can paste it directly into the file."

Then go to GitHub, open CLAUDE.md (pencil icon), paste the updates, commit.

Takes 2 minutes. Saves 20 minutes next session.

1

GitHub is your AI's brain. A repo with CLAUDE.md and skills means your AI never starts from scratch.

2

Skills capture YOUR expertise. You know things AI doesn't — CSR rules, grant processes, client quirks. Write them down once, use them forever.

3

Update after every session. The habit is more important than the content. Even one new bullet point keeps your AI current.

4

Share with your team. When your skill files live on GitHub, your colleagues can learn from them too. Knowledge compounds.
