Your AI Knowledge Base — Hands On

We've talked about tools, mechanics, and concepts. Today we build. By the end of this hour, you'll have a working AI knowledge base on GitHub — one that your AI assistant can read, learn from, and use every time you start a conversation.

The Big Idea

"If you can't explain it simply, you don't understand it well enough." — Richard Feynman

Here's the problem we're solving: every time you open a new Claude conversation, it's like talking to a brilliant stranger. They know everything about technology — but nothing about you, your project, or your domain.

Today, we fix that. We're going to give your AI a memory — stored on GitHub, accessible from anywhere, shareable with your team.

Think of it this way

You know how when a new team member joins, you spend the first week explaining how things work? CLAUDE.md is that onboarding document — except your AI reads it in 2 seconds and never forgets it.

This website IS a GitHub repository. Every session page, every resource, every guide — it's all markdown and HTML files stored on GitHub and published via GitHub Pages. No Word docs. No email attachments. One source of truth, always up to date, accessible to everyone.

Walkthrough

1
Set Up Your GitHub Account

If you already have one: Skip to Step 2.

If you don't: Go to github.com and sign up with your Dhwani email.

  • Pick a username you'll remember (your name works fine)
  • Free plan is all you need
  • Verify your email when prompted
What is GitHub?

Think of GitHub as Google Drive — but it remembers every version of every file you've ever saved. You can go back to any point in time. And it's built for collaboration — multiple people can work on the same files without overwriting each other.

2
Create Your Personal Repository

A repository (repo) is just a project folder on GitHub.

  • Click the + button (top right) → New repository
  • Name it: my-ai-workspace (or anything you like)
  • Set it to Private (only you can see it)
  • Check "Add a README file"
  • Click Create repository
What just happened?

You created a folder in the cloud. It has one file in it (README.md). That's it. Nothing scary. This folder is now your AI's brain — and right now it's empty. Let's fill it.

3
Create Your CLAUDE.md

CLAUDE.md is the most important file in your AI workflow. It tells AI who you are, what you work on, and how you like to work.

  • In your repo, click Add file → Create new file
  • Name it: CLAUDE.md
  • Paste the template below
  • Edit it to match your role and preferences
  • Click Commit changes (green button at bottom)
# CLAUDE.md — [Your Name] ## Who I Am - Role: [Your title at Dhwani RIS] - Domain: [What you work on — grants, CSR, dashboards, MIS, etc.] - Technical level: [Be honest — "non-technical PM", "comfortable with Excel", etc.] ## What I Work On - Products: [mGrant, Frappe LMS, dashboards, etc.] - Clients: [Who you serve, what they care about] - Current focus: [What you're working on this month] ## How I Like to Work - Be concise. Lead with the answer. - Don't ask me to write or edit code. - When in doubt, ask me — don't guess. - [Add your own preferences] ## Domain Knowledge I Have - [What do you know that the AI doesn't?] - [CSR Section 135 rules? Grant lifecycle? MIS report formats?] - [This is your superpower. Write it down.]
"The most important thing is to get started. Your first CLAUDE.md doesn't need to be perfect — it needs to exist." — Andrew Ng's principle, applied here
4
Create Your First Domain Skill

A skill is a cheat sheet you give your AI. Instead of explaining the same thing every conversation, you write it once and the AI references it forever.

Why skills matter

Imagine you hire a new analyst. Day one, you explain CSR rules. Day two, you explain them again. Day three, again. That's what happens without skills — your AI starts from zero every time. A skill file is the onboarding doc that makes day one the only day you explain.

Pick your area. What do you know that AI doesn't?

A

CSR compliance — Section 135 rules, MCA reporting, budget cycles

B

Grant lifecycle — proposal to disbursement to closure

C

Dashboard specs — KPIs, data sources, filter logic, RLS

D

MIS reporting — report formats, data validation, client expectations

E

Client management — communication norms, escalation paths, SLAs

F

Something else entirely — your expertise, your rules, your domain

Now create it:

  • In your repo, click Add file → Create new file
  • Name it: skills/[your-area].md (e.g., skills/csr-compliance.md)
  • Use the template below as a starting point
  • Commit when done
# Skill: [Your Area] # Purpose: Give AI the domain knowledge it needs ## What This Covers [One paragraph: what area does this skill address?] ## Key Concepts - [Concept 1]: [Brief explanation] - [Concept 2]: [Brief explanation] - [Concept 3]: [Brief explanation] ## Rules & Constraints - [What must always be true in this domain?] - [What are the compliance requirements?] - [What mistakes do people commonly make?] ## Common Tasks - [Task 1]: [How it should be done] - [Task 2]: [How it should be done] ## Vocabulary - [Term 1] = [What it means in your context] - [Term 2] = [What it means in your context]

Start small. 10 bullet points is better than an empty file. You can always add more later. The point is to capture what's in your head so your AI can use it.

5
Connect GitHub to Claude UI

Now let's make Claude actually read your GitHub repo. No terminal required.

Generate a token:

  • Go to github.com → click your avatar (top right) → Settings
  • Scroll down to Developer settings (bottom of left sidebar)
  • Click Personal access tokens → Tokens (classic)
  • Click Generate new token (classic)
  • Name: claude-access
  • Expiration: 30 days (you can always regenerate)
  • Scopes: check repo (that's all you need)
  • Click Generate token
  • Copy it immediately — you won't see it again

Use it in Claude:

  • Open claude.ai
  • Start a new conversation
  • Tell Claude: "I have a GitHub repo at github.com/[your-username]/my-ai-workspace. Here is my token: [paste token]. Please read my CLAUDE.md and skills folder, then tell me what you learned about me."
  • Watch it read your files and summarize your profile

What just happened? Claude read your GitHub files and now understands your role, your domain, and your preferences. Next conversation, paste the same instruction and it starts from the same baseline. No more explaining from scratch.

Security note: Personal access tokens are like passwords. Never share them in group chats or emails. They're for your Claude sessions only. If one leaks, revoke it immediately from GitHub Settings.

6
Update Your Repo After Every Session

The most common mistake: creating CLAUDE.md once and never updating it. Your AI's memory is only as good as the last update.

After every AI session, ask Claude:

"Summarize what we did today and give me updates for my CLAUDE.md. Format it so I can paste it directly into the file."

Then go to your GitHub repo, open CLAUDE.md via the web editor (pencil icon), paste the updates, and commit.

Takes 2 minutes. Saves 20 minutes next session.
