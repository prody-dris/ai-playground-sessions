Thursday, 30 April 2026 First open room
Architecture · A2A reframe

Most “agents talking” problems are routing problems.

We sat down to wire two agents into an A2A protocol. Twenty minutes in, the reframe: we don't actually need agents calling each other. We need both agents available in any chat — tag either, both respond. That's a routing fix, not a protocol fix.

Take it forward: before reaching for an A2A pattern, ask the cheaper question — do the agents need to hand off context, or do you just want both in the room?

Nihaan Mohammed · Ankit Jangir
Product · Ticketing MVP

The PRD is the build.

CLI scaffold up; PRD scope drove every decision. Same lesson as S5 — once the spec is crisp, the build is cheap. The hour was spent writing the brief, not the code, and that's where it should be spent.

Take it forward: if you're stuck mid-build with AI, the spec was probably soft. Don't iterate on the code — go upstream.

Akshat Verma · Kashish Talwani
Craft · A generic stack for the product

Every successful build owes a PR back to the stack.

A long sidebar between Ankit Jangir and Ankit Android, on agents and what a generic stack for the product actually looks like. The sharp idea: when AI ships a feature, ask it — “what should change in our stack so the next person doesn't hit what we hit?” Push that change back. The stack compounds. The product wouldn't, on its own.

Take it forward: make stack-improvement a standing thread. After every shipped feature, raise one stack PR — even a tiny one.

Ankit Jangir · Ankit Android
